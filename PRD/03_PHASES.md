# Phase 분리 계획

> 순서 기준: **버그 먼저 → 구조 다음 → 효과 마지막.**
> 지금 화면이 깨져 보이는 문제를 안고 새 기능을 얹으면, 무엇 때문에 어색한지 구분이 안 된다.

---

## Phase 1 — 지금 깨져 있는 것 (MVP)

새 기능 없이 현재 화면을 정상으로 만든다. 이것만 해도 체감이 크게 달라진다.

### 1-1. 모바일 줄바꿈

- [x] 히어로 슬로건에 데스크톱/모바일 줄바꿈 규칙 적용
- [x] 720px 이하에서 데스크톱 전용 줄바꿈을 띄어쓰기로 흘림
- [x] 상세 페이지의 기존 `renderRich` 규칙과 동작을 맞춤

**기준**: 360px · 720px · 1280px 세 폭에서 문구가 어긋나지 않을 것

### 1-2. 단어별 그라데이션

- [x] 선택 구간을 단어 단위 `<span class="grad">`로 분할
- [x] `data-c` 색 번호 지원
- [x] `sanitizeBreaks()` 화이트리스트에 `class` · `data-c`만 통과
- [x] 색 칩 클릭 → 선택한 단어에 적용
- [x] 기존에 저장된 구간형 span은 그대로 렌더 (마이그레이션 없음)

**기준**: 어느 단어를 봐도 그라데이션 전체 색을 가질 것 · 저장 후 새로고침해도 유지될 것

### 1-3. 성능 마무리

- [x] 남은 `backdrop-filter` 점검 (모달·기기 미리보기·드래그 손잡이)
- [x] 화면 밖 애니메이션 정지 확인

**기준**: 노트북에서 위아래 스크롤 시 끊김이 느껴지지 않을 것

---

## Phase 2 — 생각을 주인공으로 (핵심)

이 사이트를 고치는 진짜 이유. Phase 1 없이 여기부터 하면 새 섹션도 같이 어색해 보인다.

### 2-1. 생각 섹션

- [x] `thoughts[]` 데이터 모델 + Firestore 저장/불러오기
- [x] 카드 렌더 (제목 + 2~3줄 본문 + 날짜)
- [x] 편집 모드 추가·삭제·수정, 고정(pinned) 토글
- [x] 기본 4~6개 노출 + "더 보기"
- [x] 스크롤 등장 효과 (아래에서 20px 떠오름)

### 2-2. 섹션 순서 재배치

- [x] 생각 → 프로젝트 → 프로그램 순으로
- [x] 채널 · 협업 아코디언을 푸터로 흡수

### 2-3. 프로젝트 개편

- [x] `workflows[]`에 `problem` · `outcome` 추가
- [x] 문제 → 흐름 → 결과 레이아웃
- [x] `problem`을 가장 크게

**기준**: 사이트를 처음 본 사람이 스크롤 두 번 안에 "이 사람은 이런 생각을 한다"를 말할 수 있을 것

---

## Phase 3 — 효과

없어도 사이트는 제 역할을 한다. 성능 예산 안에서만 얹는다.

### 3-1. 커서 잔상

- [x] 단일 캔버스 + 알파 페이드
- [x] 셰이더 팔레트에서 색 가져오기
- [x] 데스크톱 전용, `prefers-reduced-motion` 비활성
- [x] 프레임이 밀리면 자동으로 끔

### 3-2. 다듬기

- [x] 생각 카드 ↔ 프로젝트 연결 표시 (`workflowId`)
- [x] 카드 등장 시차

**기준**: 잔상을 켠 상태로도 Phase 1의 스크롤 기준을 지킬 것

---

## 순서를 이렇게 잡은 이유

| 원칙 | 설명 |
|---|---|
| 버그가 먼저 | 깨진 화면 위에 새 기능을 얹으면 원인 구분이 안 된다 |
| 구조가 효과보다 먼저 | 커서 잔상은 아무리 예뻐도 "무슨 생각을 하는 사람인지"를 못 알려준다 |
| Phase마다 배포 가능 | 각 단계가 끝날 때마다 사이트가 온전한 상태 |

## 하지 않을 것

Phase 3까지 끝나도 손대지 않는다 — 다국어 · 검색 · 태그 필터 · RSS · 댓글.

### 남은 확인 (사용자)

- 실제 생각 글을 3~4개 써보고 카드 크기·2열 배치가 맞는지
- 모바일에서 히어로 문구가 제대로 흘러가는지
- 커서 잔상의 속도·진하기가 과하지 않은지

### 되돌리는 법

커서 잔상이 거슬리면 `#cursorTrail` 캔버스 요소 한 줄만 지우면 된다.
스크립트는 요소가 없으면 스스로 빠진다.

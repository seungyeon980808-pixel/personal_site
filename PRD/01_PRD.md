## 1. 왜 고치는가

해커톤 선정이 끝나면서 이 사이트의 목적이 바뀌었다.

- **이전**: 포트폴리오 제출용 — 심사자에게 "무엇을 만들었는가"를 증명
- **지금**: 자기소개 — 방문자에게 **"이 사람은 이런 생각을 하는구나"**를 남기는 것

현재 구조는 이전 목적에 맞춰져 있다. 프로그램 목록이 화면의 주인공이고, 생각은 히어로 슬로건 두 줄이 전부다. 방문자는 *무엇을 만들었는지*는 알 수 있지만 *왜 만드는 사람인지*는 알 수 없다.

**해결하려는 문제: 사이트가 "만든 것"만 보여주고 "생각"을 못 보여준다.**

---

## 2. 성공 기준

방문자가 페이지를 닫을 때 이 세 가지가 남아야 한다.

1. 이 사람은 **요즘 이런 주제에 빠져 있다**
2. 이 사람은 **이런 문제를 불편해하고, 그래서 이런 걸 만든다**
3. (부수적) 만든 것들의 목록

지금은 3번만 전달된다.

## 4. 기능 요구사항

### 4.1 생각 섹션 (신설) — 최우선

사이트의 새 주인공. 히어로 바로 아래.

| 항목 | 내용 |
|---|---|
| 단위 | 카드 하나 = 짧은 글 하나 (2~3줄) |
| 주기 | 주 1~2회 갱신 |
| 구성 | 한 줄 제목 + 두세 줄 본문 |
| 작성 | 사이트에서 직접 (관리자 로그인 → 편집 모드) |
| 정렬 | 최신순, 고정(pinned) 항목은 맨 앞 |
| 노출 | 기본 4~6개, 그 아래는 "더 보기" |

**등장 효과**: 스크롤 시 아래에서 20px 정도 부드럽게 떠오름. 기존 `.reveal` 패턴 재사용.

### 4.2 프로젝트 개편 — 문제 → 흐름 → 결과

현재는 흐름도(화살표로 잇는 프로그램)만 있다. 앞뒤에 두 칸을 추가한다.

```
[문제]  시험 문제 하나 내는 데 저녁이 다 간다        ← 크게, 주인공
   ↓
[흐름]  ExamPool → 5E → HwpPalette                ← 현행 유지
   ↓
[결과]  소재 찾기부터 조판까지 한 번에 끝난다        ← 한 줄
```

`workflows` 배열에 `problem` · `outcome` 두 필드를 추가한다.

### 4.3 단어별 그라데이션

**현재 문제**: 구간 전체를 `<span class="grad">` 하나로 감싸서 그라데이션 하나를 늘여 바른다. 그래서 첫 단어는 파랑, 끝 단어는 보라가 된다.

**해결**: 단어마다 span을 나눈다. 각 span이 자기 배경 상자를 가지므로 **모든 단어가 그라데이션 전체를 온전히 갖는다.** 여기에 단어별 색 지정을 얹는다.

```html
<span class="grad" data-c="0">교사가</span>
<span class="grad" data-c="2">한가하면</span>
```

- `data-c` = 팔레트 색 번호 (0~4)
- 편집 모드에서 단어를 드래그하고 색 칩을 누르면 그 단어에만 적용
- 여러 단어를 드래그하면 각 단어가 개별 span으로 쪼개져 같은 색을 받음

> ⚠️ 가정 2: 색 팔레트는 기존 5종(기본·아이스·선셋·민트·로즈) 유지 — 아니라면 알려주세요

### 4.4 모바일 줄바꿈

**현재 문제**: 히어로 슬로건이 데스크톱 기준으로 `<br>`이 박혀 있어 모바일에서 문구가 어긋난다.

**해결**: 상세 페이지에 이미 있는 규칙을 히어로에도 적용한다 (`CLAUDE.md` 문서화됨).

- 엔터 1번 = **데스크톱 전용 줄바꿈** → 720px 이하에서는 띄어쓰기로 흘림
- 엔터 2번(빈 줄) = 문단 구분 → 어느 폭에서든 유지

새 규칙을 만들지 않고 기존 관례를 재사용한다.

### 4.5 커서 잔상

마우스를 따라 **배경 셰이더와 같은 계열 색**이 번졌다 사라지는 잔상.

- 단일 캔버스에 알파 페이드로 그린다 (요소를 여러 개 두지 않는다)
- 색은 셰이더 팔레트에서 가져와 배경과 이어져 보이게
- 데스크톱 전용 — 모바일은 커서가 없다
- `prefers-reduced-motion`이면 비활성

> ⚠️ 가정 3: 커서 잔상은 데스크톱에서만 — 아니라면 알려주세요

## 5. 성능 예산

**기준: 일반 노트북에서 스크롤이 끊기지 않을 것.**

이미 처리한 것

- `.section`/`.now`의 `backdrop-filter` 제거 (움직이는 셰이더 위에서 매 프레임 블러 재계산)
- 셰이더 해상도 제한 (`minPixelRatio 1`, `maxPixelCount 140만`)

앞으로 지킬 것

| 규칙 | 이유 |
|---|---|
| 전체 화면 애니메이션 레이어는 **최대 2장** (셰이더 + 커서 잔상) | 합성 비용이 장수에 비례 |
| 움직이는 배경 위에 `backdrop-filter` 금지 | 프레임마다 블러 재계산 |
| 스크롤 등장은 `transform` + `opacity`만 | 레이아웃 재계산 없음 |
| 화면 밖 애니메이션은 정지 | `IntersectionObserver` |

효과가 과하면 자동으로 줄인다 — 프레임이 밀리면 커서 잔상부터 끈다.
